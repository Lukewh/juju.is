Coffee on K8s —
Live from London

Human operators have spoken: Kubernetes must make coffee! Due to popular demand, we built a vendor-agnostic espresso machine controlled by a server, running in a kubernetes cluster. Scroll down to learn more about this project and brew a cup.

The project

Earlier this year we asked, “What can’t you do with Kubernetes?”. The most voted answer was… make coffee.

Yes, the beverage. “Haha, funny” — we thought to ourselves, until we realized that this is an amazing opportunity to tell a story about multi-cloud usage, open source IoT stacks and electronics!

TL;DR: We built a coffee machine controlled by a microcontroller talking to a server running on K8s. It can be deployed on public and private clouds. Here’s the stack:

Overwhelmed? That’s ok, we broke it down each part in a series of blog posts.

The idea here is to, well, make some coffee, but most importantly: show that you can deploy and operate an open source stack of apps that will work equally well on any kubernetes flavour or vendor or cloud platform using Juju and Charmed Operators.

Live from London

We planned to have this machine running live at Kubecon NA. However, because of health and safety regulations, we had to quarantine our coffee machine at Canonical’s office in London. Worry not, you can still place an order, and see it being executed (sort of). Yes, it also makes tea!
